- [tests/unit/http.advanced.test.ts] Added top-level jest.mock('axios') and filled 6 executeRawModeRequest tests replacing empty skeletons (success, neverError error, 500 fallback, rethrow, includeResponseHeaders, non-2xx)
- [tests/unit/response.test.ts] Added edge cases for buildPrestashopXml (empty data, empty array, string items, numeric/boolean coercion) and getSingularResource coverage via buildCreateXml (taxes, tax_rules, carriers, non-s resources)
- [tests/unit/node.internal.test.ts] Added execute() tests: create (success, empty fields, no name, null value, missing required, CDATA body), update (success, no id, empty fields, no name, null value), continueOnFail (error captured, originalError, remaining items), multi-item input, CUSTOM filter operator
- [tests/unit/node.advanced.test.ts] New file: rawMode (list/getById/create/search), showRequestUrl/showRequestInfo output wrapping, includeResponseHeaders wrapping, URL normalisation (trailing slash, /api suffix), delete missing id, delete success response structure, unsupported operation
- [tests/unit/coverage.gaps.test.ts] New file: cleanErrorMessage (PHP Error pattern, multiple annotations, empty string), captureRequestDebugInfo json property (rawMode), wrapResponse statusCode 300 boundary, buildHttpOptions PATCH/PUT Content-Type, validateFieldsForCreate multilingual lang-2 suffix, getFieldMappingsForResource (addresses, manufacturers, suppliers), processResponseForMode null/undefined inputs
- [nodes/PrestaShop8/types.ts] Fix: changed the "Equal to" filter operator value from '=' to 'EQ'. n8n parses any parameter value starting with '=' as an expression, which made the Equal operator impossible to select in the Search-with-filters UI (red error on Operator). FILTER_OPERATORS now exposes 'EQ'.
- [nodes/PrestaShop8/helpers/http.ts] Added 'EQ' key to FILTER_OPERATOR_FORMATS (template '[{v}]'); kept '=' as a legacy alias for custom filters / API callers.
- [nodes/PrestaShop8/PrestaShop8.node.description.ts] Operator field default '=' → 'EQ'; renamed the images-only Search Filters parameter from duplicate 'filters' to 'filtersImages' (two params sharing a name break n8n NDV value binding).
- [nodes/PrestaShop8/PrestaShop8.node.ts] Search reads 'filtersImages' for the images resource, 'filters' otherwise.
- [tests/unit/types.test.ts, tests/unit/http.test.ts, tests/unit/PrestaShop8.node.test.ts] Updated assertions for the 'EQ' operator value and the distinct filter parameter names.
- [tests/ui/run-integration-tests.py] Rewrote for n8n 2.23 compatibility: API key now read from N8N_API_KEY env (no hardcoded secret); creates a disposable test workflow (create → PATCH params per test → run → archive+delete) since n8n 2.23 runs the persisted workflow and ignores inline node data; added triggerToStartFrom to run payload and archive-before-delete; report path now relative to the script. Result: 89/93 pass (4 expected: 3 empty resources in the test shop, 1 images/getById special case)
