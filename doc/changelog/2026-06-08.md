# Changelog - 2026-06-08

## Summary
Test coverage analysis: fixed 6 failing/incorrect tests, corrected a null-handling bug in xml.ts, and added 70 new tests covering 11 confirmed gaps.

## Changes

- [tests/unit/coverage.gaps.test.ts] Fixed rawMode delete `deletedId` type assertion (string vs number); replaced 3 incorrect `getAvailableFields` marker tests (schema has all flags false) with correct behavior assertions; added imports for `buildUpdateXml` and `convertResourceArray`; added tests for `buildCreateXml`/`buildUpdateXml` with SINGULAR_RESOURCE_MAP resources, `convertResourceArray` direct coverage, `_id` suffix field conversion, and `wrapResponse(false)` explicit path
- [tests/unit/node.advanced.test.ts] Added tests for `execute()` create (normal flow, empty fields, missing required, rawMode, images resource), update (normal flow, empty fields, missing id, empty field name), search non-rawMode (filter URL, no filters throws, CUSTOM operator, IS_EMPTY operator, images filtersImages param), continueOnFail, and list with advancedOptions.dateFormat
- [nodes/PrestaShop8/utils/xml.ts] Fixed null guard in buildFieldsXml: condition now excludes null in addition to undefined, preventing null.toString() crash
- [tests/unit/final-coverage-gaps.test.ts] Fixed 2 test assertions for null/undefined field values to match actual behavior (fields are silently skipped)
- [tests/unit/new-coverage-gaps.test.ts] Added 34 tests covering: execute() create/update null value throws, search dateFormat param, buildUrlWithFilters with 0 and rawMode, convertValue float path, escapeXml special chars, processDisplayParameter custom without fields, validateFieldsForCreate multilingual stripping, getSingularResource fallback, whitespace-only field name

- [nodes/PrestaShop8/loadOptions.ts] Use optional catch binding (`catch {}`) in two handlers where the error was unused — clears ESLint warnings
- [nodes/PrestaShop8/utils/response.ts] Prefixed unused `resource` parameters with `_` in simplifyItem and convertSimplifiedToPrestaShop — clears ESLint warnings
