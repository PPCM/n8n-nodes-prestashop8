# Changelog - 2026-06-09

## Summary
Release 2.4.2: ship the null-guard fix in XML field building.
Release 2.5.0: new option to throttle PrestaShop calls with a configurable delay between input items.

## Changes

- [nodes/PrestaShop8/utils/xml.ts] Skip fields whose value is `null` (not only `undefined`) in buildFieldsXml, preventing a `null.toString()` crash during Create/Update operations.
- [package.json] Bump version to 2.4.2, then 2.5.0 for the delay option.
- [nodes/PrestaShop8/PrestaShop8.node.description.ts] Added "Delay Between Calls (ms)" option (Options collection) to throttle requests.
- [nodes/PrestaShop8/helpers/http.ts] Added `sleep()` helper and `delayBetweenCalls` to OperationOptions / getOperationOptions.
- [nodes/PrestaShop8/PrestaShop8.node.ts] Apply the delay before each PrestaShop call except the first (between input items).
- [tests/unit/http.test.ts] Added tests for `sleep` and `getOperationOptions` delay reading.
- [tests/unit/identified-gaps.test.ts] New test file covering 6 previously uncovered branches: delayBetweenCalls multi-item sleep, showRequestInfo fallback for delete, images resource customField-only path, continueOnFail=true error recovery, circular reference fallback in extractPrestashopError, and processResponseForMode third-condition array scan.
