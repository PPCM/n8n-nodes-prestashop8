# Changelog - 2026-06-11

## Summary
New "Retry On Error" option: automatically retry a PrestaShop call that fails on a transient error (network timeout, connection drop, 5xx server error or 429 rate-limit). The retry budget is per failing call (reset for each call), with a configurable max attempts and fixed delay between attempts.

## Changes

- [nodes/PrestaShop8/helpers/http.ts] Added `RetryOptions` interface, `isRetryableError()` (network codes + 429 + 5xx, never 4xx) and `withRetry()` (per-call retry loop, rethrows after exhausting budget). Wrapped the JSON (`executeHttpRequest`) and raw XML (`executeRawModeRequest`) calls with `withRetry`. Added `retry` to `OperationOptions` / `getOperationOptions`.
- [nodes/PrestaShop8/PrestaShop8.node.description.ts] Added "Retry On Error" collection (Enabled, Max Retries, Retry Delay) to the Options collection.
- [nodes/PrestaShop8/PrestaShop8.node.ts] Pass `opts.retry` to every HTTP call site (list, getById, create, update, search, delete; JSON and raw modes).
- [tests/unit/http.test.ts] Added tests for `isRetryableError`, `withRetry` (success, disabled, retry-then-succeed, exhaust budget, non-retryable) and retry option reading in `getOperationOptions`.
- [package.json] Bump version to 2.6.0 for the Retry On Error option, then 2.6.1 for retry logging + tooltip.
- [.github/workflows/publish.yml] Bump actions/checkout v4→v5 and actions/setup-node v4→v5 (Node.js 20 deprecation on GitHub runners).
- [nodes/PrestaShop8/helpers/http.ts] Added `describeError()` helper and an optional `onRetry` hook to `RetryOptions`/`withRetry` to report each retry attempt.
- [nodes/PrestaShop8/PrestaShop8.node.ts] When retry is enabled, log each attempt to the n8n server logs via `this.logger.warn` (retry N/max, resource/operation, error reason, delay).
- [nodes/PrestaShop8/PrestaShop8.node.description.ts] Moved retry-reasons info from a notice block into the "Enabled" field tooltip (the `?` hover).
- [.github/workflows/publish.yml] Disable setup-node package-manager cache (pnpm not installed on the runner, which uses npm ci).
- [README.md] Document the Retry On Error option (feature list, Advanced Options, Timeout troubleshooting).

## Notes
With `Never Error` enabled in Raw mode, 5xx/429 responses are returned as-is without retry (axios does not throw); network/timeout errors are still retried. Default behaviour is unchanged when retry is disabled (`maxRetries=0` → identical to previous code).
