# Changelog - 2026-06-27

## Summary
Fixed the search filter behavior where a value-requiring operator (e.g. `=`/`EQ`) with an empty value was silently dropped, causing PrestaShop to return all records instead of an empty/filtered set.

## Changes

- [nodes/PrestaShop8/PrestaShop8.node.ts] Added a guard in the search filter builder: a `requiresValue` operator with an empty value now throws an explicit NodeOperationError suggesting the `IS_EMPTY` operator, instead of being silently skipped
- [tests/unit/node.internal.test.ts] Added a test asserting that a value-requiring operator with an empty value throws and never sends the request
- [tests/unit/final-coverage-gaps.test.ts] Updated the former "skip empty value" test to assert the new throw behavior

## Notes
Root cause: in the filter loop, the condition `!format.requiresValue || filterValue` skipped the filter entirely when the value was empty, producing an unfiltered request. Users wanting to match empty fields must use the `IS_EMPTY` operator (`filter[field]=[]`).
